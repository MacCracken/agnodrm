## Findings

### F-11 ‚Äî `update_check` writes the NUL terminator one byte past a full read buffer

**Severity:** HIGH (heap overflow on attacker-influenced input).

`update_check` (`src/update.cyr`) read a manifest into `alloc(16384)` with a loop
that capped `total` at 16384, then wrote `store8(buf + total, 0)` ‚Äî at offset
16384, one byte past the 16384-byte allocation, whenever the manifest is ‚â•16KB.
Manifests come from a `file://` URL (and, later, network fetch), i.e. external
input.

**Fix:** reserve the last byte for the terminator (read `16384 - total - 1`, cap
`total` at `16384 - 1`), mirroring the already-correct `ima_read_measurements`
idiom. Regression test `big_check_*` writes an 18KB manifest and asserts a clean
parse. (Note: a 1-byte overflow is not reliably caught without a sanitizer; the
fix is verified by construction + the proven-safe sibling pattern.)

### F-12 ‚Äî `update_save_state` builds JSON into a fixed 512-byte buffer with no bounds

**Severity:** MEDIUM (overflow from a corrupt/hostile state file).

`update_save_state` wrote `version` and `pending` fields ‚Äî which can originate
from a loaded on-disk state file ‚Äî into a fixed `alloc(512)` via unbounded
`memcpy`, with no length guard. A long field value overflows the allocation.

**Fix:** size the buffer from the actual variable-length fields
(`160 + vlen + plen`, where fixed scaffolding totals <120 bytes), so the
allocation always fits its contents.

### F-13 ‚Äî `ima_read_measurements` silently truncates the IMA log at 64KB

**Severity:** MEDIUM (attestation completeness ‚Äî a trust-library gap).

The IMA measurement log was read into a fixed 64KB buffer and silently truncated
beyond it. On a busy system the log exceeds 64KB, so attestation would parse an
incomplete measurement list ‚Äî a malicious or post-compromise entry past the 64KB
mark would be invisible.

**Fix:** grow the buffer to EOF (start 64KB, double on fill) with a 32MB safety
ceiling that returns `Err` on a runaway/malformed log instead of silently
dropping the tail.

### F-14 ‚Äî `netns` subprocess execution was non-functional and used bare command names

**Severity:** LOW (latent ‚Äî consumers use only nftables rendering today; fix adds
PATH-hijack hardening).

Every `ip`/`mkdir`/`nft` invocation in `src/netns.cyr` passed a hand-built argv
array as a single argument to `run_capture`/`_exec3`, which only supports
`cmd + 2` args ‚Äî so `ip netns add <name>` never reached `ip`. The commands were
also bare names (`"ip"`), which `execve` cannot resolve (no PATH search), and
which would be PATH-hijackable for a privileged caller even if they did.

**Fix:** rewrote all 15 exec sites to build a cstr `vec` and run it through
`exec_vec`/`exec_capture` (the proven `journald` pattern), with **absolute**
command paths (`/usr/sbin/ip`, `/usr/sbin/nft`, `/usr/bin/mkdir`); the inner
command after `ip netns exec` stays bare (PATH-resolved by `ip` itself).
Regression test `exec_vec_multiarg` proves multi-arg absolute-path exec end to
end (the privileged `ip` path needs `CAP_NET_ADMIN`, so it can't run in CI). The
capability map now correctly records netns's subprocess binaries.

## Refactor / optimization (Tier 1‚Äì3)

Driven by the same audit; behavior-preserving except the F-11‚ÄìF-14 fixes.

**Tier 1 ‚Äî cross-module dedup (new `src/util.cyr`, `agnosys_*` helpers, in
`[lib.core]` so every profile bundle resolves them; public names kept as thin
wrappers so the frozen API surface is preserved):**
- `agnosys_json_emit_cstr_or_null` ‚Äî replaced 5 byte-identical
  `_<mod>_emit_cstr_or_null` shims (mac, dmverity, drm, certpin, update).
- `agnosys_is_hex_char` ‚Äî single source for the hex predicate (ima, dmverity
  public wrappers; tpm + update inline checks, removing a mixed `&&`/`||`).
- `agnosys_cstr_starts_with` ‚Äî single-pass prefix check (fuse, udev wrappers).
- `agnosys_run_capture` / `agnosys_run_checked` ‚Äî shared subprocess wrappers
  (dmverity, luks, tpm), error string passed per-tool.
- *Deferred:* `*_is_name_char` (dmverity, luks) is only 2 instances ‚Äî below the
  3rd-instance extraction threshold; left in place.

**Tier 2 ‚Äî bench-gated hot-path optimizations** (vs. the 6.0.24 baseline):
- `agnosys_cstr_starts_with` single-pass: `starts_with_hit` **47 ‚Üí 16ns**
  (‚àí66%), `starts_with_miss` **31 ‚Üí 7ns** (‚àí77%).
- `mac_default_profile` builds straight into the heap alloc (dropping a static
  staging buffer + an extra copy + a `var buf[N]` static-data hazard):
  **~325 ‚Üí ~227ns** (‚àí30%).
- `tpm_get_random` sizes its hex buffer to `count*2+8` (‚â§ the old fixed 8200)
  and drops a dead `memset` (not benchmarked ‚Äî execs an external tool).

**Tier 3 ‚Äî hygiene:**
- Removed `break` from `var`-declaring `while` loops (audit.cyr, mac.cyr) ‚Äî a
  documented hard-constraint violation.
- `journald` filter argv now uses the generated `journald_filter_*` accessors
  instead of raw `load64(filter + N)` offsets.
- `audit_open` emits a real `log_warn` instead of an empty `if` block.
- *Deferred (API-breaking ‚Äî belong in a major, not 1.3.0):* removing the unused
  `agnosys_checked_syscall` (public, 0 callers, dup of `wrap_syscall`) and the
  unused `label` parameter of `dmverity_validate_hex`.
